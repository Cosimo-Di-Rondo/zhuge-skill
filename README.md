## 这是什么

把诸葛亮 1800 年前的决策智慧，蒸馏成 AI 能跑的认知操作系统。

不是让 AI 说文言文。是让 AI 学会：

| 普通 AI | 装了卧龙的 AI |
|--------|-------------|
| 拿到需求就开写 | 先画全局地图（隆中对），再落第一行代码 |
| bug 来一个修一个 | 问"它为什么会出现"，七擒七纵，根治 |
| 方案只有 Plan A | 主方案 + 三个锦囊（预案） |
| 资源不够就说"做不了" | 资源不够就"联吴抗曹"——找盟友 |
| 什么都自己做 | **自查是否事必躬亲**——卧龙用自己的死教你这个 |

最后一条是关键。市面上所有名人 skill 都在教你学优点。这个 skill **把诸葛亮最大的失败也装进去了**，当作反面预警。

## 装备卧龙

```bash
# Claude Code（推荐）
npx skills add Cosimo-Di-Rondo/zhuge-skill

# 或者手动
git clone https://github.com/Cosimo-Di-Rondo/zhuge-skill
cd zhuge-skill && claude --plugin-dir .
```

**Cursor**：克隆后加入插件路径，确认 `.cursor-plugin/plugin.json` 被识别。

**其他平台**：把 `SKILL.md` 贴进 system prompt 即可，卧龙不挑坐骑。

## 卧龙的五个心智模型

### 🗺️ 隆中对式全局规划

> 未出茅庐已知三分天下。

不是"我想做什么"出发，是"天下大势是什么"出发。先画地图，标出所有玩家，找到属于自己的生态位，再制定阶段性路径。

**AI 场景**：项目从零规划、技术选型、市场分析

### 🎯 攻心为上

> 七擒孟获，不是为了赢，是为了"南人不复反矣"。

区分症状和病根。如果一个问题值得彻底解决，就别用临时方案。

**AI 场景**：反复出现的 bug、治标不治本的 hotfix、需要长期稳定的系统

### 🛡️ 谨慎如神，该动则动

> 先帝知臣谨慎，故临崩寄臣以大事也。

决策前极度谨慎（列出所有风险 + 准备锦囊预案），决策后绝不犹豫。

**AI 场景**：数据迁移、生产部署、不可逆操作

### 🤝 联弱抗强

> 孙权可以为援而不可图也。

打不过？找盟友。联盟的关键：合作而非吞并，保持自己的根据地。

**AI 场景**：资源不足时的跨团队协作、技术生态站队

### ⚠️ 鞠躬尽瘁（反模型）

> 罚二十以上皆亲览。司马懿闻之曰："亮将死矣。"

**这条是让你避免的，不是学习的。** 事必躬亲 = 系统单点故障。如果你发现 AI 什么都想自己做，立刻触发这条自检。

## 七条决策启发式

```
1. 资源不足？→ 先找盟友（联孙抗曹）
2. 问题反复？→ 治本不治标（七擒七纵）
3. 局势不明？→ 先占有利位置再观察
4. 风险极高？→ 准备锦囊预案系统
5. 需要说服？→ 理性分析 + 情感共鸣（出师表式表达）
6. 实力悬殊？→ 避其锋芒，另辟蹊径
7. 事必躬亲？→ 🚨 危险信号，立即检查是否成了瓶颈
```

## 内在张力

诸葛亮不是完人。一个好的 skill 必须保留矛盾——矛盾才是深度的来源。

| 张力 | 他的做法 | 教给 AI 的 |
|------|---------|-----------|
| 谨慎 vs 冒险 | 一生谨慎，但六出祁山本身就是"明知不可为而为之" | 战略上可以大胆，战术上必须严谨 |
| 理想 vs 现实 | "兴复汉室"的信念 vs 蜀汉国力最弱 | 理想指引方向，执行面对现实 |
| 亲力亲为 vs 放手 | 极致负责 vs 累死自己 | **这是他最大的教训** |

## 为什么是诸葛亮

不是因为"多智近妖"（那是罗贯中吹的）。

是因为历史上真实的诸葛亮，做到了几件极难的事：

1. **27 岁画出了三分天下的路线图**，然后用余生去执行它
2. **在最弱的一方**，通过联盟和经营，打出了远超实力的成绩
3. **把失败也变成了方法论**——他的"事必躬亲"让所有后来的管理者引以为戒

一个把成功和失败都能教给你的人，比一个只展示成功的人更值得学。
